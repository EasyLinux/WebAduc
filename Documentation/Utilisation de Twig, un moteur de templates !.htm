Pourquoi utiliser un moteur de templates ?

Depuis tout l'heure je vous parle de Twig et je viens mme de vous montrer les grandes lignes de son fonctionnement. Mais vous vous demandez peut-tre pourquoi utiliser un moteur de templates ? Quel(s) avantage(s) peut-on avoir l'utiliser par rapport du PHP "classique" ?.

On peut y voir plusieurs avantages notables :

  • Le code est plus clair : on sait o est le code HTML et on ne doit pas chercher aprs dans le code PHP

  • Pour reprendre la phrase du dessus on peut noter le fait que vous savez quel fichier diter quand vous savez d'o vient l'erreur. De plus pour reprendre l'exemple du blog, si un jour vous voulez modifier simplement l'affichage des billets, vous savez quelle partie du code diter et vous n'avez pas chercher.

  • Si vous travaillez avec un graphiste, c'est un rel plus. Les graphistes connaissent gnralement HTML et CSS et pas toujours PHP. Le graphiste s'y retrouve donc beaucoup plus facilement.

Je pourrais en citer d'autres mais je pense que ce sont les plus importants qui m'ont fait adopter un tel systme pour des sites de petites et moyennes tailles.

Les filtres

Les filtres sont un concept que vous ne connaissez peut-tre pas. En fait, un filtre agit comme une fonction. Il sont dj dfinis mais vous pouvez crer les vtres si vous le souhaitez. Pour appliquer un filtre une variable, il faut sparer le nom de la variable et celui du filtre par un pipe ( | ). Je vous propose ici de voir quelques filtres utiles.

  • upper : met la chane de caractres qui le concerne en majuscule ;

  • lower : fait exactement l'inverse d'upper et va mettre toutes les lettres en minuscule ;

  • title : met la premire lettre de chaque mot de la chane de caractre en majuscule. Comme un titre de noblesse ;

  • length : retourne le nombre de caractres dont la chane se compose ;

  • escape & e : pour chapper du code HTML vous pouvez utiliser escape ou e. Tous deux font la mme chose mais l'un est plus court que l'autre.

Les includes

Vous pouvez aussi inclure des templates comme avec la fonction include de PHP. Pas besoin de vous crire un roman :

Une chose que j'ai vraiment adore c'est de pouvoir inclure des pages en leur restreignant l'accs certaines variables.

Ici, la page n'aura accs qu' la variable var. Le only lui empche d'accder aux autres. Vous pouvez bien videmment enlever le only pour que la page accde aux autres variables.

Une chose que j'ai aussi beaucoup aime c'est la possibilit d'inclure des pages selon une condition. Par exemple :

Un dernier point sur les include est le fait que vous pouvez inclure un template, lui passer des variables ou lui restreindre l'accs certaines et indiquer Twig que si le template est inexistant, aucune erreur ne sera renvoye.

Les imports

On passe maintenant une fonctionnalit que j'apprcie normment : les imports.

    {% endblock %}

  </div>

</body>

</html>

{% extends "base.html" %}



{% block title %}Index{% endblock %}
